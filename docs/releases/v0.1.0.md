# qqnotify-go v0.1.0

`v0.1.0` is the first formal public release of `qqnotify-go`.

## Highlights

- Public Go client for QQ notifications
- Reusable templates for Codex, CI, and cron notifications
- Lightweight HTTP bridge for non-Go callers
- Optional bridge auth with Bearer tokens
- Health check endpoint for deployments
- Example programs and public-facing documentation

## Included

- `qqnotify.LoadConfigFromEnv`
- `qqnotify.NewClient`
- `qqnotify.NewClientWithOptions`
- `qqnotify.NewCodexNotification`
- `qqnotify.NewCINotification`
- `qqnotify.NewCronNotification`
- `cmd/qqnotifyd`
- `cmd/example-send`

## Notes

- This is a `v0.x` release. The repository is usable and documented, but the API may still evolve before `v1.0.0`.
- `qqnotifyd` supports plain `title/body` payloads and template-aware payloads.
- Template payloads currently support:
  - `type=codex`
  - `type=ci`
  - `type=cron`
